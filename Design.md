1. Color System & Usage
This defines the foundational palette and assigns functional roles to each color to ensure clarity in your interface.

Primary Red (Vibrant Red): This is your high-energy, action-oriented color. Use it to draw immediate attention.

Main Action Buttons (CTAs): Use for crucial steps like "Ship Now," "Track Package," or "Confirm."

Crucial Alerts: Use for errors or critical warnings that require user attention.

Primary Blue (Deep Corporate Blue): This is your foundational, trustworthy color. Use it to establish stability and structure.

Headers & Navigation: Apply this to the main app header, side menus, and tab navigation bars.

Links and Interactive States: Use this color for hyperlinks and secondary buttons (e.g., "View Past Shipments").

Background (White/Light Grey): Keep backgrounds clean and light to allow the red and blue to stand out without competing for attention.

2. General Interface Guidelines
Hierarchy: Reserve red for the most important single action on any screen. If everything is highlighted, nothing is.

Color Ratio: Aim for a "blue-dominant" interface with strategic pops of red. This keeps the experience professional and reliable rather than frantic.

Text and Contrast:

Always prioritize readability.

Use dark grey or black for main body text (never use blue or red for bulk reading).

Ensure high contrast when text is placed on a color (e.g., use white text on the blue header or red buttons). Avoid placing red text on blue (or vice versa).

3. Iconography & Motifs
Logo Integrity: Always use the full logo where possible (like the loading screen or settings page).

Brand Motif: The abstract, angular shapes of the logo icon can be adapted into subtle design motifs, like subtle gradients or background watermark patterns, to reinforce the brand identity without being overly explicit.

Color Context: Maintain consistency in small UI elements. For example, use blue for progress indicators and standard system icons (map, clock).
